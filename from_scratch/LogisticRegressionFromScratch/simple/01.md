Вот код для **логистической регрессии** (бинарная классификация), написанный в том же стиле, что и ваш код для линейной регрессии:

```python
# %%
import numpy as np
import matplotlib.pyplot as plt

# Сделаем так, чтобы можно было воспроизвести результат
np.random.seed(42)

# Сгенерируем 100 точек для БИНАРНОЙ КЛАССИФИКАЦИИ
m = 100  # количество примеров

# Генерируем X от 0 до 10
X = np.random.rand(m, 1) * 10

# Создаем два класса:
# Класс 1: точки, где y = 1 (x > 5 + шум)
# Класс 0: точки, где y = 0 (x < 5 - шум)
y = np.zeros((m, 1))
for i in range(m):
    if X[i] > 5 + np.random.randn() * 1.5:
        y[i] = 1
    elif X[i] < 5 + np.random.randn() * 1.5:
        y[i] = 0

# Посмотрим на данные
plt.figure(figsize=(6, 4))
plt.scatter(X[y.flatten() == 0], y[y.flatten() == 0], color='blue', alpha=0.7, label='Класс 0')
plt.scatter(X[y.flatten() == 1], y[y.flatten() == 1], color='red', alpha=0.7, label='Класс 1')
plt.xlabel("X")
plt.ylabel("Класс (0 или 1)")
plt.title("Данные для бинарной классификации")
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()

# %%
# Инициализируем веса случайными числами (очень маленькими)
w = np.random.randn(1) * 0.01
b = np.zeros(1)

print(f"Начальный вес (w): {w[0]:.4f}")
print(f"Начальное смещение (b): {b[0]:.4f}")

# %%
def sigmoid(z):
    """
    Сигмоидная функция (логистическая функция)
    Превращает любое число в вероятность от 0 до 1
    """
    return 1 / (1 + np.exp(-z))

def compute_loss(y_pred_prob, y_true):
    """
    Считаем бинарную кросс-энтропию (Binary Cross-Entropy / Log Loss)
    Это стандартная функция потерь для логистической регрессии
    """
    m = y_true.shape[0]
    # Добавляем маленькое число (1e-15), чтобы избежать log(0)
    epsilon = 1e-15
    y_pred_prob = np.clip(y_pred_prob, epsilon, 1 - epsilon)
    
    loss = -(1 / m) * np.sum(
        y_true * np.log(y_pred_prob) + (1 - y_true) * np.log(1 - y_pred_prob)
    )
    return loss

def train_step(X, y, w, b, learning_rate=0.01):
    """
    Один шаг обучения логистической регрессии (Forward + Backward)
    """
    m = X.shape[0]
    
    # --- 1. Forward pass (Прямой проход) ---
    # Линейная комбинация (как в линейной регрессии)
    z = np.dot(X, w) + b
    
    # Применяем сигмоиду, чтобы получить ВЕРОЯТНОСТИ (от 0 до 1)
    y_pred_prob = sigmoid(z)
    
    # --- 2. Считаем ошибку (чтобы видеть прогресс) ---
    loss = compute_loss(y_pred_prob, y)
    
    # --- 3. Backward pass (Обратный проход) ---
    # Производная бинарной кросс-энтропии по z (перед сигмоидой):
    # dL/dz = y_pred_prob - y_true
    # Это КЛЮЧЕВОЕ отличие от линейной регрессии!
    dz = y_pred_prob - y
    
    # Градиенты через цепное правило
    dw = (1 / m) * np.dot(X.T, dz)
    db = (1 / m) * np.sum(dz)
    
    # --- 4. Обновляем веса (Градиентный спуск) ---
    w = w - learning_rate * dw
    b = b - learning_rate * db
    
    return w, b, loss

# %%
# Гиперпараметры
learning_rate = 0.1  # Для логистической регрессии можно брать побольше
epochs = 2000

# Списки для истории
loss_history = []
w_history = []
b_history = []

print("Начинаем обучение логистической регрессии...")
print("---")

for epoch in range(epochs):
    w, b, loss = train_step(X, y, w, b, learning_rate)
    
    loss_history.append(loss)
    w_history.append(w[0])
    b_history.append(b[0])
    
    if epoch % 200 == 0:
        print(f"Эпоха {epoch:4d} | Потеря (Loss): {loss:.4f} | w = {w.flatten()[0]:.4f} | b = {b.flatten()[0]:.4f}")

print("---")
print(f"ИТОГИ обучения:")
print(f"Финальный вес (w): {w.flatten()[0]:.4f}")
print(f"Финальное смещение (b): {b.flatten()[0]:.4f}")

# %%
# Визуализация результатов
plt.figure(figsize=(12, 4))

# График 1: Данные и разделяющая линия
plt.subplot(1, 2, 1)
plt.scatter(X[y.flatten() == 0], y[y.flatten() == 0], color='blue', alpha=0.6, label='Класс 0')
plt.scatter(X[y.flatten() == 1], y[y.flatten() == 1], color='red', alpha=0.6, label='Класс 1')

# Рисуем сигмоиду (вероятность) и порог 0.5
x_line = np.linspace(0, 10, 100)
# Для каждого x считаем вероятность принадлежности к классу 1
z_line = w[0] * x_line + b[0]
y_prob_line = sigmoid(z_line)

plt.plot(x_line, y_prob_line, color='green', linewidth=3, label='Вероятность P(y=1|x)')
plt.axhline(y=0.5, color='black', linestyle='--', alpha=0.5, label='Порог 0.5')

plt.xlabel("X")
plt.ylabel("Вероятность класса 1")
plt.legend()
plt.title("Логистическая регрессия")
plt.grid(True, alpha=0.3)

# График 2: Как уменьшалась ошибка (Loss) со временем
plt.subplot(1, 2, 2)
plt.plot(loss_history)
plt.xlabel("Эпоха")
plt.ylabel("Binary Cross-Entropy Loss")
plt.title("Сходимость градиентного спуска")
plt.grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

# %%
# Точность модели
y_pred_prob = sigmoid(np.dot(X, w) + b)
y_pred_class = (y_pred_prob >= 0.5).astype(int)
accuracy = np.mean(y_pred_class == y) * 100

print(f"Точность модели: {accuracy:.2f}%")
```

## Ключевые отличия от линейной регрессии:

| Компонент | Линейная регрессия | Логистическая регрессия |
|-----------|-------------------|------------------------|
| **Что предсказываем** | Числовое значение (например, цену) | Вероятность (от 0 до 1) |
| **Выходной слой** | Линейный: `y_pred = X*w + b` | Сигмоида: `p = sigmoid(X*w + b)` |
| **Функция потерь** | MSE (среднеквадратичная) | Binary Cross-Entropy (логарифмическая) |
| **Градиент dw** | `(2/m) * X.T * (y_pred - y)` | `(1/m) * X.T * (p - y)` |
| **Градиент db** | `(2/m) * sum(y_pred - y)` | `(1/m) * sum(p - y)` |
| **Интерпретация** | Линия, проходящая через данные | Разделяющая граница между классами |

---

## Почему градиенты отличаются?

**В линейной регрессии (MSE):**
```
dL/dy_pred = 2*(y_pred - y)
```

**В логистической регрессии (Cross-Entropy + Sigmoid):**
```
dL/dz = p - y   (где p = sigmoid(z))
```

Магия в том, что производная **сигмоиды** и производная **кросс-энтропии** сокращаются, давая простую и красивую формулу:
```
p = sigmoid(z)
dL/dz = p - y
```

## Как интерпретировать результат?

- **`w > 0`**: Чем больше X, тем выше вероятность класса 1
- **`w < 0`**: Чем больше X, тем выше вероятность класса 0
- **Порог 0.5**: Точка, где модель не может выбрать класс (50/50)

В нашем случае, так как мы сгенерировали данные с разделением около X = 5, модель найдёт `w` и `b` такие, что `w*x + b = 0` примерно при `x = 5`.
