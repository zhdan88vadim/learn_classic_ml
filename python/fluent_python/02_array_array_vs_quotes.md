Отличный вопрос! Действительно, зачем использовать `array.array`, если можно просто сделать список `[]`?

Ответ: **`array.array` и список (`[]`) — это принципиально разные вещи**, и выбор зависит от того, что вам нужно.

---

## Главное отличие: что они хранят?

### Список `[]`
```python
my_list = [36, 162, 163, 165, 8364, 164]
```
- Хранит **ссылки** на объекты Python.
- Каждое число — это **отдельный объект** в памяти.
- Для числа `8364` создается объект `int` (который занимает ~28 байт), для числа `162` — еще один объект, и т.д.

**Сколько памяти занимает список из 6 чисел?**
- Сам список: ~56 байт (служебная информация) + 6 ссылок по 8 байт = ~104 байта
- 6 объектов `int`: 6 × 28 байт = 168 байт
- **Итого: ~272 байта**

---

### Массив `array('I')`
```python
import array
my_array = array.array('I', [36, 162, 163, 165, 8364, 164])
```
- Хранит **сырые C-шные значения** (как в языке C).
- Числа хранятся **компактно**, как последовательность байтов, без отдельных объектов.

**Сколько памяти занимает массив из 6 чисел (тип 'I' = 4 байта на число)?**
- Сам массив: ~64 байта (служебная информация) + 6 × 4 байта = 24 байта
- **Итого: ~88 байт**

---

### Сравнение памяти для 1 миллиона чисел

| Структура | Память (приблизительно) |
| :--- | :--- |
| Список `[]` | ~36 МБ |
| Массив `array('I')` | ~4 МБ |

**Разница в 9 раз!**

## Когда использовать `array.array`, а когда `[]`?

| Ситуация | Что использовать | Почему |
| :--- | :--- | :--- |
| Нужно хранить **миллионы чисел** и экономить память | `array.array` | Память критична, массив в разы компактнее |
| Нужно хранить **разные типы** (числа, строки, объекты) | `[]` | Список может хранить что угодно, массив — только числа одного типа |
| Нужно часто **изменять размер** (добавлять/удалять элементы) | `[]` | Список оптимизирован для этого, массив тоже умеет, но медленнее |
| Нужно **быстро** обращаться к элементам по индексу | `array.array` | Скорость доступа к сырым данным выше (но разница не всегда заметна) |
| Нужно работать с **бинарными файлами** или **сетевыми протоколами** | `array.array` | Можно легко сохранить/загрузить как байты |
| Нужно использовать **математические операции** (numpy) | `numpy.array` | Но это уже другая история |

## Пример, где `array` спасает жизнь

Представьте, что вы обрабатываете **аудиофайл** или **изображение** — там миллионы чисел (отсчеты звука, пиксели).

```python
# Плохо: список съест всю память
samples_list = [0] * 10_000_000  # 10 млн нулей → ~280 МБ!

# Хорошо: массив сэкономит память
from array import array
samples_array = array('I', [0]) * 10_000_000  # → ~40 МБ
```

---

## Но для вашего примера с 6 символами...

Для 6 чисел разница в памяти (272 байта против 88 байт) — **несущественна**. 

В учебном примере авторы используют `array.array` чтобы:
1. **Показать синтаксис** с генераторными выражениями.
2. **Показать**, что генератор можно передавать в разные функции, не только в `tuple()` и `list()`.

**На практике** для 6 чисел никто не станет заморачиваться с `array`. Написали бы просто:
```python
my_list = [ord(symbol) for symbol in symbols]
```

## Итог

| Вопрос | Ответ |
| :--- | :--- |
| Зачем `array.array`, если есть `[]`? | Для **экономии памяти** при работе с большими объемами однотипных чисел. |
| Когда использовать `[]`? | Для большинства повседневных задач, когда чисел не миллионы. |
| Когда использовать `array.array`? | Когда память критична (встроенные системы, обработка сигналов, большие данные) или нужно взаимодействовать с C/C++ библиотеками. |

**Простое правило:** Если не знаете, что выбрать — выбирайте `[]`. Он удобнее, гибче и быстрее для маленьких объемов. Про `array` вспоминайте, когда программа начнет падать с ошибкой `MemoryError`.
