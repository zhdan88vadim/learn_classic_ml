Отличный вопрос! Вы абсолютно правы: в PEP 572 сказано, что переменная, созданная через `:=` (моржовый оператор), по умолчанию принадлежит **текущей локальной области видимости** (обычно это функция). 

Но если вы заранее объявите эту переменную как `global` или `nonlocal`, то оператор `:=` будет изменять именно **внешнюю** переменную, а не создавать локальную.

Давайте посмотрим на конкретные примеры, чтобы понять разницу.

---

### Пример 1: Обычное поведение (локальная переменная)
По умолчанию `:=` создает переменную внутри функции.

```python
def test_local():
    # Здесь НЕТ global/nonlocal
    x = 10  # Внешняя переменная (не видна внутри функции напрямую)
    
    # Используем :=, чтобы создать НОВУЮ локальную переменную b
    if (b := x + 5) > 10:
        print(f"Локальная b внутри функции: {b}")
    
    # Попробуем напечатать b снаружи функции (ошибка, её нет в глобальной области)
    # print(b) <-- Ошибка! NameError: name 'b' is not defined

test_local()
# print(b) <-- Тоже ошибка!
```
**Что происходит:** `b` создается **только внутри** функции. Снаружи функции этой переменной не существует.

---

### Пример 2: Использование `global` (меняем глобальную переменную)
Если мы хотим, чтобы `:=` изменил переменную на уровне всего скрипта, мы объявляем её как `global`.

```python
b = 100  # Глобальная переменная

def test_global():
    global b  # Говорим Python: "Не создавай локальную b, используй глобальную!"
    
    x = 10
    # Сейчас := изменит глобальную b с 100 на 15
    if (b := x + 5) > 10:
        print(f"Значение b внутри функции (изменено): {b}")

test_global()
print(f"Значение b СНАРУЖИ функции: {b}")  # Выведет 15!
```
**Что происходит:** Строка `global b` заставляет Python искать переменную `b` не внутри функции, а в самом верху (глобально). Поэтому `:=` перезаписал глобальное значение `100` на `15`.

---

### Пример 3: Использование `nonlocal` (меняем переменную внешней функции)
Это используется, когда у вас есть **вложенные функции** (функция внутри функции). `nonlocal` говорит: "Не создавай новую переменную, а используй ту, что в функции-родителе".

```python
def outer_function():
    b = 50  # Переменная внутри внешней функции
    
    def inner_function():
        nonlocal b  # Говорим: "Используй b из outer_function, а не создавай свою!"
        
        x = 10
        # := изменит переменную b из внешней функции
        if (b := x + 5) > 10:
            print(f"b внутри inner_function (изменено): {b}")
    
    inner_function()
    print(f"b внутри outer_function (после изменения): {b}")  # Выведет 15!

outer_function()
```
**Что происходит:** Без `nonlocal` оператор `:=` создал бы переменную `b` только внутри `inner_function`. Но так как мы написали `nonlocal`, он нашел `b` в функции-родителе (`outer_function`) и изменил её с `50` на `15`.

---

### Короткий итог для запоминания:

| Код внутри функции | Что делает `:=` |
| :--- | :--- |
| **Нет** `global`/`nonlocal` | Создает **новую локальную** переменную (не влияет на внешний мир). |
| Есть `global b` | Изменяет **глобальную** переменную (на уровне файла). |
| Есть `nonlocal b` | Изменяет **переменную внешней функции** (при вложенных функциях). |

**Важное дополнение:** В PEP 572 также сказано, что если переменная была объявлена через `global` или `nonlocal` **в любой точке** текущей функции, то `:=` будет работать с ней как с внешней, даже если объявление стоит ниже по коду.
